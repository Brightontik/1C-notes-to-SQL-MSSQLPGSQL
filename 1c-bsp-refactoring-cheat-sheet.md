# Памятка: Справочник API БСП против прикладных «Велосипедов»

Этот документ содержит реестр типовых антипаттернов («велосипедов») в коде 1С, их замену на стандартные методы БСП и архитектурное обоснование: почему БСП работает быстрее, стабильнее и эффективнее с точки зрения O-нотации, RAM и СУБД.

## ⚡ Реестр оптимизации функций и процедур

### 1. Чтение одного реквизита объекта

*   **❌ Наивный «велосипед»:**
    ```bsl
    Вид = СсылкаНоменклатуры.ВидНоменклатуры;
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    Вид = ОбщегоНазначения.ЗначениеРеквизитаОбъекта(СсылкаНоменклатуры, "ВидНоменклатуры");
    ```
*   **🧠 Архитектурное обоснование:**
    При обращении через точку платформа 1С выполняет `SELECT` *всех* полей объекта из базы данных и кеширует весь объект целиком в RAM сеанса. Метод БСП строит точечный SQL-запрос `SELECT ВидНоменклатуры FROM ...`, извлекая строго нужные данные. Сложность по памяти снижается с **O(M)** (размер объекта) до **O(1)**.

---

### 2. Пакетное чтение нескольких реквизитов

*   **❌ Наивный «велосипед»:**
    ```bsl
    Арт = Ссылка.Артикул;
    Наим = Ссылка.Наименование;
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    СтруктураРеквизитов = ОбщегоНазначения.ЗначенияРеквизитовОбъекта(Ссылка, "Артикул, Наименование");
    ```
*   **🧠 Архитектурное обоснование:**
    Два обращения через точку (если объект успел вытесниться из оперативной памяти) — это два раздельных обращения к серверу БД. Метод БСП делает ровно **один** пакетный запрос, забирая все требуемые поля за один сетевой раундтрип к СУБД.

---

### 3. Проверка существования ссылки в БД

*   **❌ Наивный «велосипед»:**
    ```bsl
    ОбъектСуществует = (Ссылка.ПолучитьОбъект() <> Неопределено);
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    ОбъектСуществует = ОбщегоНазначения.СсылкаСуществует(Ссылка);
    ```
*   **🧠 Архитектурное обоснование:**
    `ПолучитьОбъект()` — это максимально дорогая операция. Платформа поднимает в RAM сервера приложений все данные, табличные части и инициирует модули. Метод БСП делает сверхлегкий `SELECT 1 FROM Таблица WHERE Ссылка = @Параметр`. База данных отвечает за **O(1)** без выделения памяти под объект.

---

### 4. Динамическое чтение констант в циклах

*   **❌ Наивный «велосипед»:**
    ```bsl
    Для Каждого Стр Из ТЧ Цикл
        Если Константы.УчетПоСкладам.Получить() Тогда // Спам СУБД
            // Логика
        КонецЕсли;
    КонецЦикла;
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    УчетВключен = СтандартныеПодсистемыПовтИсп.ПараметрыРаботыПрограммы().УчетПоСкладам;
    Для Каждого Стр Из ТЧ Цикл
        Если УчетВключен Тогда // Чтение из RAM за O(1)
            // Логика
        КонецЕсли;
    КонецЦикла;
    ```
*   **🧠 Архитектурное обоснование:**
    Каждый вызов `Получить()` константы внутри цикла порождает отдельный SQL-запрос к БД (сложность **O(N)**). Метод БСП берет значение из глобального кэша параметров работы программы в RAM текущего рабочего процесса `rphost` за стабильные **O(1)**.

---

### 5. Поиск ссылок по коду или наименованию

*   **❌ Наивный «велосипед»:**
    ```bsl
    Валюта = Справочники.Валюты.НайтиПоКоду("643");
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    Валюта = ОбщегоНазначения.СправочникОбъектПоРеквизиту("Справочник.Валюты", "Код", "643");
    ```
*   **🧠 Архитектурное обоснование:**
    Стандартный поиск платформы каждый раз выполняет новый `SELECT` к базе. Метод БСП под капотом использует кэшируемые модули с повторным использованием. Если валюта "643" уже искалась в этом сеансе, СУБД вообще не затрагивается — значение мгновенно возвращается из памяти.

---

### 6. Программная модификация реквизитов объекта

*   **❌ Наивный «велосипед»:**
    ```bsl
    Об = Ссылка.ПолучитьОбъект();
    Об.Статус = НовыйСтатус;
    Об.Записать();
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    ОбщегоНазначения.ЗаписатьРеквизитыОбъекта(Ссылка, Новый Структура("Статус", НовыйСтатус));
    ```
*   **🧠 Архитектурное обоснование:**
    Обычная запись объекта вызывает *все* подписки на события, тяжелые контроли в `ПередЗаписью` и перезаписывает все табличные части. БСП делает оптимизированную модификацию только указанных колонок через установку флага `ОбменДанными.Загрузка = Истина` на уровне платформы, снижая нагрузку на CPU сервера в десятки раз.

---

### 7. Преобразование строки JSON в структуру

*   **❌ Наивный «велосипед»:**
    ```bsl
    Чтение = Новый ЧтениеJSON;
    Чтение.УстановитьСтроку(Стр);
    // Парсинг без контроля закрытия потока
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    Данные = ОбщегоНазначения.ЗначениеИзСтрокиJSON(Стр);
    ```
*   **🧠 Архитектурное обоснование:**
    Самописный парсинг больших строк часто оставляет открытыми дескрипторы при возникновении исключений, вызывая утечки RAM. БСП гарантирует атомарное чтение в блоках `Попытка - Исключение` с обязательным гарантированным освобождением памяти в блоке финализации.

    ---

### 8. Получение значений дополнительных реквизитов (Свойства)

*   **❌ Наивный «велосипед»:**
    ```bsl
    // Запрос в цикле или соединение с тяжелой таблицей ТЧ "ДополнительныеРеквизиты"
    Запрос = Новый Запрос;
    Запрос.Текст = 
        "ВЫБРАТЬ Значение ИЗ Справочник.Номенклатура.ДополнительныеРеквизиты 
        |ГДЕ Ссылка = &Ссылка И Свойство = &Свойство";
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    Значение = УправлениеСвойствами.ЗначениеСвойства(СсылкаНоменклатуры, СвойствоСсылка);
    ```
*   **🧠 Архитектурное обоснование:**
    Самописный запрос каждый раз обращается к табличной части СУБД. Подсистема управления свойствами БСП использует оптимизированный внутренний кэш и ПВЗ-модули. Если свойства объекта уже запрашивались в текущем контексте, БСП отдает их из RAM за **O(1)**, полностью исключая избыточный I/O на дисковую подсистему.

---

### 9. Установка управляемой блокировки по ссылке

*   **❌ Наивный «велосипед»:**
    ```bsl
    Блокировка = Новый БлокировкаДанных;
    ЭлементБлокировки = Блокировка.Добавить("Документ.РеализацияТоваровУслуг");
    ЭлементБлокировки.УстановитьЗначение("Ссылка", СсылкаДок);
    Блокировка.Заблокировать();
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    ОбщегоНазначения.ЗаблокироватьДанные(СсылкаДок);
    ```
*   **🧠 Архитектурное обоснование:**
    В «велосипеде» легко ошибиться в имени пространства блокировок при рефакторинге (особенно при переименовании метаданных), что приведет к скрытому пропуску блокировки в транзакции. Метод БСП сам вычисляет тип метаданных по переданной ссылке или массиву ссылок, строит корректное пространство блокировки платформы и защищает от `Deadlocks` на уровне СУБД.

---

### 10. Формирование строки со списком кодов для SQL-инъекций

*   **❌ Наивный «велосипед»:**
    ```bsl
    СтрокаКодов = "";
    Для Каждого Код Из МассивКодов Цикл
        СтрокаКодов = СтрокаКодов + "'" + Код + "',"; // O(N^2) по памяти из-за перевыделения строк
    КонецЦикла;
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    СтрокаКодов = СтрокаФункцииКлиентСервер.СтрокаИзМассиваСубтитров(МассивКодов, ","); 
    // Или СтрокаИменДляЗапроса
    ```
*   **🧠 Архитектурное обоснование:**
    Конкатенация строк в цикле 1С порождает создание нового объекта строки в памяти на каждом шаге, что для больших массивов уходит в **O(N²)** по времени и забивает RAM. БСП использует встроенные оптимизированные механизмы работы со строковыми буферами платформы, сводя сложность к линейной **O(N)**.

---

### 11. Безопасное чтение структуры из JSON/XML с проверкой свойств

*   **❌ Наивный «велосипед»:**
    ```bsl
    Если СтруктураДанных.Свойство("Имя") И СтруктураДанных.Имя <> Неопределено Тогда
        Значение = СтруктураДанных.Имя.Значение; // Риск падения, если Имя не структура
    КонецЕсли;
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    Значение = ОбщегоНазначения.СвойствоСтруктуры(СтруктураДанных, "Имя.Значение", АльтернативноеЗначение);
    ```
*   **🧠 Архитектурное обоснование:**
    «Велосипедные» многоуровневые проверки вложенности объектов (Dot-notation в JSON) раздувают код и часто приводят к рантайм-ошибкам `Поле объекта не обнаружено`. БСП реализует безопасный обход дерева структуры по указанному пути за **O(1)** по коду, гарантируя стабильность потока без перегрузки логики блоками `Попытка...Исключение`.

---

### 12. Генерация уникального GUID без деградации индексов

*   **❌ Наивный «велосипед»:**
    ```bsl
    // Платформа создает случайный UUID (v4)
    НовыйУИД = Новый УникальныйИдентификатор(); 
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    // БСП генерирует последовательный UUID, оптимизированный под B-Tree индексы СУБД
    НовыйУИД = ОбщегоНазначения.УникальныйИдентификатор() 
    ```
*   **🧠 Архитектурное обоснование:**
    Случайные GUID при вставке в таблицы СУБД (например, в качестве ключа регистров) вызывают жесткую фрагментацию индексов (Page Splits). Метод БСП создает *последовательный (sequential)* идентификатор. При массовой записи новые строки ложатся строго в конец страниц индекса. Нагрузка на дисковую подсистему СУБД при записи падает в разы.

---

### 13. Проверка: Открыта ли транзакция в текущем потоке

*   **❌ Наивный «велосипед»:**
    ```bsl
    // Разработчик пытается использовать глобальные переменные 
    // или флаги, чтобы понять, находится ли он внутри НачатьТранзакцию()
    Если ВТранзакции Тогда ...
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    ВнутриТранзакции = ОбщегоНазначения.ВТранзакции();
    ```
*   **🧠 Архитектурное обоснование:**
    Попытка управлять флагами транзакций вручную ломается при первом же аварийном исключении (когда база выполняет `Rollback`). Метод БСП напрямую опрашивает контекст выполнения платформы через низкоуровневый API, гарантируя стопроцентную точность статуса транзакции без накладных расходов на хранение ложных состояний в памяти сеанса.

### 14. Сравнение двух таблиц значений на равенство

*   **❌ Наивный «велосипед»:**
    ```bsl
    // Вложенные циклы: O(N * M). Смерть производительности при больших таблицах.
    Для Каждого Стр1 Из ТЗ1 Цикл
        Для Каждого Стр2 Из ТЗ2 Цикл
            // Сравнение колонок...
        КонецЦикла;
    КонецЦикла;
    ```

    Провокация рантайм-исключений ради проверки прав — худший паттерн, парализующий работу CPU сервера приложений 1С из-за накладных расходов на обработку прерываний виртуальной машины. БСП опрашивает флаг профиля безопасности сеанса напрямую из свойств ядра за **O(1)**.

### 32. Безопасное получение структуры метаданных объекта по имени

*   **❌ Наивный «велосипед»:**
    ```bsl
    // Тяжелый обход коллекции метаданных платформы
    МетаданныеОбъекта = Метаданные.НайтиПоПолномуИмени("Документ.РеализацияТоваровУслуг"); 
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    МетаданныеОбъекта = ОбщегоНазначения.МетаданныеОбъектаByПолноеИмя("Документ.РеализацияТоваровУслуг");
    // Метод использует внутренний кэш метаданных БСП
    ```
*   **🧠 Архитектурное обоснование:**
    Прямой метод `Метаданные.НайтиПоПолномуИмени()` при каждом вызове сканирует дерево конфигурации, что при больших размерах метаданных (например, в ERP) создает заметную нагрузку на CPU. БСП кеширует карту соответствия имен и объектов метаданных при старте, отдавая дескриптор из памяти за **O(1)**.

### 33. Программная генерация паролей или хэшей высокой энтропии

*   **❌ Наивный «велосипед»:**
    ```bsl
    // Генерация на основе генератора случайных чисел (ГСЧ), который выдает предсказуемые ряды
    ГСЧ = Новый ГенераторСлучайныхЧисел;
    Пароль = Строка(ГСЧ.СлучайноеЧисло(100000, 999999));

*   **✅ Стандарт БСП:**

    СлучайнаяСтрока = ОбщегоНазначения.СлучайнаяСтрока(ДлинаПароля);
    ```
*   **🧠 Архитектурное обоснование:**
    Стандартный `ГСЧ` 1С инициализируется от системного таймера и генерирует псевдослучайные числа, уязвимые для криптоанализа. БСП задействует криптографически стойкие алгоритмы операционной системы, обеспечивая максимальную энтропию и криптостойкость генерируемых токенов авторизации без утечек памяти.

### 34. Оптимизированный поиск дублей в массиве структур

*   **❌ Наивный «велосипед»:**
    ```bsl
    // Перебор массива массивов со сложностью O(N^2)
    Для Каждого Стр1 Из Массив Цикл
        Для Каждого Стр2 Из Массив Цикл
            Если Стр1.Кличка = Стр2.Кличка Тогда ...
        КонецЦикла;
    КонецЦикла;
    ```
*   **✅ Стандарт БСП:**
    ```bsl
    Дубликаты = ОбщегоНазначенияКлиентСервер.НайтиДубликатыВМассиве(Массив);
    ```
*   **🧠 Архитектурное обоснование:**
    Вместо ресурсоемких вложенных циклов БСП сортирует входной массив или загоняет его элементы во внутреннюю хэш-таблицу. Это гарантирует падение вычислительной сложности поиска повторяющихся элементов с катастрофической квадратичной **O(N²)** до строго линейной **O(N)**.
