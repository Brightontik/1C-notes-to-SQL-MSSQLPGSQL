# Оформление текстов запросов (Стандарт 1С #std437)

**Область применения:** Управляемое приложение, мобильное приложение, обычное приложение.
**Цель правила:** Обеспечить читаемость кода, устойчивость алгоритмов к рефакторингу метаданных и корректную работу автоматических проверок (АПК / Сонар).

---

## 1. Регистр ключевых слов

Все ключевые слова языка запросов должны быть написаны **ЗАГЛАВНЫМИ** буквами.

*❌ Неправильно:*
```bsl
Выбрать Ссылка Из Справочник.Номенклатура Где ПометкаУдаления = Ложь
```

*✅ Правильно:*
```bsl
ВЫБРАТЬ Ссылка ИЗ Справочник.Номенклатура ГДЕ ПометкаУдаления = ЛОЖЬ
```

---

## 2. Псевдонимы полей и источников (Устойчивость кода)

Рекомендуется явно указывать даже необязательные конструкции запроса. Каждому выбираемому полю и источнику необходимо назначать осмысленный псевдоним.

### 2.1. Явные псевдонимы для полей составного типа и реквизитов
Если в алгоритме используется обращение по имени свойства к выборке, имя поля в запросе должно быть жестко зафиксировано через `КАК`.

> 💡 **Пример:** Если поле объявлено как `Касса.Валюта`, при изменении имени реквизита в метаданных придется переписывать и код чтения выборки. Если указать `Касса.Валюта КАК Валюта`, то изменение имени реквизита потребует правки только текста запроса.

### 2.2. Псевдонимы полей-ссылок других полей
Особое внимание следует уделять автоматически присваиваемым псевдонимам для полей вида `Касса.Валюта.Наименование`. По умолчанию поле получит имя `Наименование`, что вызовет конфликт при выборе нескольких наименований.
* **Правило:** Присваивать псевдоним по цепочке связей, например: `ВалютаНаименование`.

### 2.3. Обязательность ключевого слова `КАК`
Перед псевдонимом поля или источника **обязательно** указывать ключевое слово `КАК`.

*❌ Неправильно:*
```bsl
ВЫБРАТЬ
	Товары.Номенклатура,
	Товары.Количество
ИЗ
	Документ.РеализацияТоваровУслуг.Товары КАК Товары
```

*✅ Правильно:*
```bsl
ВЫБРАТЬ
	Товары.Номенклатура КАК Номенклатура,
	Товары.Количество КАК Количество
ИЗ
	Документ.РеализацияТоваровУслуг.Товары КАК Товары
```

---

## 3. Структурирование и форматирование текста

* Текст запроса должен быть структурирован.
* **Запрещено** писать запрос в одну строку (даже короткий). Каждый логический блок (ВЫБРАТЬ, ИЗ, ГДЕ, СГРУППИРОВАТЬ) пишется с новой строки с отступами.

---

## 4. Комментирование сложных запросов

В запросах, сложных для понимания (где используются вложенные запросы, объединения или соединения), **рекомендуется вставлять комментарии**.
* Комментарии должны объяснять, для чего используется та или иная таблица в соединении/объединении.
* ⚠️ **Важно:** Помните, что при использовании стандартного Конструктора Запросов 1С все комментарии внутри текста запроса **удаляются автоматически без предупреждения**.

---

## 5. Программная сборка и модификация запросов

При создании объекта `Запрос` в коде рекомендуется указывать комментарий: для получения какой информации или каких целей используется данный запрос.

### 5.1. Пошаговая сборка
При программной "сборке" текста запроса (через конкатенацию строк или `СтрЗаменить`) рекомендуется комментировать все этапы его сборки.

### 5.2. Доступность для Конструктора
Необходимо стремиться к тому, чтобы каждая отдельная формируемая часть запроса (или итоговый текст) могла быть открыта с помощью Конструктора Запросов.
* Это позволяет осуществить экспресс-проверку корректности синтаксиса.
* Это упрощает разработку и сопровождение кода сторонними разработчиками.

---

## 6. Примеры программной модификации (Шаблоны)

Ниже приведены официальные примеры правильной динамической модификации текстов запросов.

### Изменение имени поля выборки или таблицы

*❌ Неправильно (Ломает работу конструктора и читаемость):*
```bsl
ТекстЗапроса =
"ВЫБРАТЬ
| Номенклатура.Наименование  КАК Наименование ,
| Номенклатура. " + ИмяПоляКод + " КАК КодАртикул
|ИЗ
| Справочник.Номенклатура КАК Номенклатура";
```

*✅ Правильно (Рекомендуемый метод фирмы 1С):*
```bsl
ТекстЗапроса =
"ВЫБРАТЬ
| Номенклатура.Наименование  КАК Наименование,
| &ИмяПоляКод  КАК КодАртикул
|ИЗ
| Справочник.Номенклатура КАК Номенклатура";

ТекстЗапроса = СтрЗаменить(ТекстЗапроса, "&ИмяПоляКод", "Номенклатура." + ИмяПоляКод);
```

### 6.1. Динамическое управление помещением во временную таблицу

**Правило:** Запрещено использовать комментарии (например, `// ПОМЕСТИТЬ`) для условного включения конструкций запроса. Текст запроса по умолчанию должен быть валидным для Конструктора запросов, а ненужные части при необходимости должны вырезаться или заменяться.

*❌ Неправильно (Ломает конструктор запросов, если закомментировать ключевое слово):*
```bsl
ТекстЗапроса =
"ВЫБРАТЬ

|	Контрагенты.Ссылка КАК Ссылка
|// ПОМЕСТИТЬ втКонтрагенты
|ИЗ

|	Справочник.Контрагенты КАК Контрагенты";

Если ВыгрузитьВоВременнуюТаблицу Тогда
	ТекстЗапроса = СтрЗаменить(ТекстЗапроса, "// ПОМЕСТИТЬ", "ПОМЕСТИТЬ");
КонецЕсли;
```

*✅ Правильно (Запрос изначально валиден, при необходимости секция ПОМЕСТИТЬ удаляется):*
```bsl
ТекстЗапроса =
"ВЫБРАТЬ
|	Контрагенты.Ссылка КАК Ссылка
|ПОМЕСТИТЬ втКонтрагенты

|ИЗ
|	Справочник.Контрагенты КАК Контрагенты";

Если Не ВыгрузитьВоВременнуюТаблицу Тогда
	ТекстЗапроса = СтрЗаменить(ТекстЗапроса, "ПОМЕСТИТЬ втКонтрагенты", "");
КонецЕсли;
```

### 6.2. Конкатенация нескольких текстов запросов в пакет

**Правило:** При динамической сборке пакета запросов символ разделителя `;` должен добавляться только вместе с телом условного запроса. Нельзя оставлять знак `;` в конце строки «базового» или «условного» запроса, если после него может идти конкатенация, так как это нарушает синтаксис при невыполнении условий.

*❌ Неправильно (Символ `;` зашит внутрь условия, при невыполнении условия синтаксис пакета сломается):*
```bsl
ТекстЗапроса = " ";
Если ИспользоватьУпаковки Тогда
	ТекстЗапроса =
	"ВЫБРАТЬ

	|	Упаковки.Ссылка КАК Ссылка
	|ИЗ
	|	Справочник.Упаковки КАК Упаковки;

	|/////////////////////////////////////////////////////////////
	|";
КонецЕсли;

ТекстЗапроса = ТекстЗапроса +
"ВЫБРАТЬ

|	Номенклатура.Ссылка КАК Ссылка
|ИЗ
|	Справочник.Номенклатура КАК Номенклатура";
```

*✅ Правильно (Разделитель пакета `;` конкатенируется обособленно и только при выполнении условия):*
```bsl
ТекстЗапроса = " ";
Если ИспользоватьУпаковки Тогда
	ТекстЗапроса =
	"ВЫБРАТЬ

	|	Упаковки.Ссылка КАК Ссылка
	|ИЗ
	|	Справочник.Упаковки КАК Упаковки";

	ТекстЗапроса = ТекстЗапроса +
	"

	|;
	|/////////////////////////////////////////////////////////////
	|";
КонецЕсли;

ТекстЗапроса = ТекстЗапроса +
"ВЫБРАТЬ

|	Номенклатура.Ссылка КАК Ссылка
|ИЗ
|	Справочник.Номенклатура КАК Номенклатура";
```
